## Рекомендуемый график

### День 1: База и Списки (5-6 часов)
Сначала пройдите все **Easy** задачи из тем 1, 2, 3. Это займет около 2 часов и даст вам уверенность.
После этого переходите к **Medium** задачам по **Связным спискам** (тема 2). Обратите особое внимание на задачу **19. Remove Nth Node** — она идеально показывает, как работают двойные указатели в Go. Обязательно напишите решение с использованием `dummy` узла, чтобы избежать `nil`-паники.

### День 2: Массивы и Скользящее окно (5 часов)
Сосредоточьтесь на теме **1**. Начните с понимания паттерна "два указателя" на примере задачи **11. Container With Most Water**. Затем переходите к **3Sum** (15) — это классика, которую спрашивают на 90% собеседований.

### День 3: Динамическое программирование и Стек (5 часов)
Начните с простых задач на стек (**20. Valid Parentheses**). В ДП не пытайтесь выучить всё — сконцентрируйтесь на паттернах: одномерный массив (House Robber, 198) и задача на матрицы (Unique Paths, 62).

### День 4 (если есть время): Деревья и Графы (4 часа)
BFS/DFS — важнейшая тема. Начните с легких задач на деревья (104, 226), затем решите **102. Binary Tree Level Order Traversal** — это шаблон, который пригодится для 80% задач на деревья.
