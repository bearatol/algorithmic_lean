# KODA.md — Инструкции для работы с проектом

## Обзор проекта

Данный проект представляет собой **репозиторий для подготовки к алгоритмическим собеседованиям на языке Go**. Проект содержит две версии плана подготовки (v1 и v2), а также директорию для личных экспериментов (my).

**Версия Go:** 1.26.1

**Назначение:** Практика решения алгоритмических задач с LeetCode для успешного прохождения технических собеседований.

## Сборка и запуск

### Запуск тестов

```bash
# Запустить все тесты во всём проекте
go test ./...

# Запустить тесты для конкретной темы (v1)
go test ./01_arrays_strings_sliding_window/...

# Запустить тесты для конкретной темы (v2)
go test ./01_array_string_two_pointers/...

# Запустить тесты для конкретной задачи с детальным выводом
go test -v ./01_arrays_strings_sliding_window/001_two_sum/...
```

### Компиляция и запуск

```bash
# Запустить файл конкретной задачи
go run v1/01_arrays_strings_sliding_window/001_two_sum/main.go

# Запустить личные эксперименты
go run my/main.go
```

### Установка зависимостей

```bash
go mod download
```

## Правила разработки

- **Язык:** Go 1.26.1
- **Стиль кода:** Стандартный Go (используйте `gofmt` для форматирования)
- **Тестирование:** Каждая задача должна иметь тесты в файле `main_test.go`
- **Наименование:** Используйте snake_case для директорий (например, `001_two_sum`)
- **Комментарии:** Добавляйте ссылку на LeetCode и сложность в начале файла

---

## Часто используемые паттерны

- **Два указателя (Two Pointers):** Для массивов и строк
- **Скользящее окно (Sliding Window):** Для подмассивов
- **BFS/DFS:** Для деревьев и графов
- **Динамическое программирование:** 1D и 2D массивы
- **Hash Map:** Для поиска дубликатов и группировки
